第十二章：性能优化

Warps和SIMD

执行过程：

warps：

一个线程块由 32 个线程组成warps（线程束） warps是在多处理器上以物理方式并行(SIMD)执行的

warps作为调度单位

每个块被分成32-线程的warps

  • 一种实现技术，不是 CUDA 编程模型的一部分
  • Warps 是 SM 中的调度单元
  • Warp 中的线程以单指令多数据 (SIMD) 方式执行
  • warps中的线程数在未来几代可能会有所不同

多维线程块中的warps

线程块首先按行主要顺序线性化为一维，首先是 x 维度，然后是 y 维度，最后是 z 维度。

线性化后对块进行分区

线性化线程块被分区

  • warps内的线程索引是连续的并且不断增加
  • Warp 0 从线程 0 开始

跨设备分区方案一致

  • 因此，您可以在控制流中使用这些知识
  • 但是，warps的确切大小可能会代代相传

不要依赖warps内或warps之间的任何排序

  • 如果线程之间存在任何依赖关系，则必须__syncthreads() 以获得正确的结果（稍后会详细介绍）。

SM 是 SIMD 处理器

指令获取、解码和控制的控制单元在多个处理单元之间共享，控制开

控制分支

warps中的线程遇到控制分支时，通过做出不同的控制决策，采取不同的控制流路径

  • 一些采用 then 路径，另一些采用 if 语句的 else 路径
  • 一些线程与其他线程采用不同数量的循环迭代

在当前的 GPU 中，采取不同路径的线程的执行是被序列化的（串行的）。

  • 一个warps中的线程所采用的控制路径一次遍历一个，直到不再存在。
  • 考虑嵌套控制流语句时，不同路径的数量可能很大

控制分支示例

当分支或循环条件是线程索引的函数时，可能会出现分支 具有分支的内核语句示例： if (threadIdx.x > 2) { }

  • 这为块中的线程创建了两个不同的控制路径

  • 决策粒度 < warps大小；在第一个warp中，线程 0、1 和 2 遵循与其余线程不同的路径

没有发散的例子： If (blockIdx.x > 2) { }

  • 决策粒度是块大小的倍数；任何给定warps中的所有线程都遵循相同的路径

设备代码：

分析向量大小有1000个元素： 假设块大小为 256 个线程，每个块中有 8 个warp

  1. Block0、1 和 2 中的所有线程都在有效范围内。 i 值从 0 到 767，这三个Block有24个经线，没有一个会有控制分支。

假设 16x16 tiles和线程块，每个线程块有 8 个warps (256/32) 假设 100x100 的方阵，每个warp将经历 7 个阶段（上限为 100/16） 有 49 个线程块（每个维度 7 个）

实际上：

有42（6x7）个类型1块，总共有336（8x42）条warp 它们都有 7 个阶段，因此有 2,352 (336*7) 个warp-phases warps只有在最后阶段才有控制分支，336 个wrap-phases有控制分支

类型2块：分配加载底部瓦片的7个块，共56（8x7）个warp 它们都有 7 个阶段，所以有 392 (56x7) 个warp-phase 每个类型 2 块中的前 2 个warp（16x4/32）将保持在有效范围内，直到最后一个阶段。 剩余的 6 个warp不在有效范围内 所以，只有 14 (2*7) 个warp-phasse有控制分支

控制分支的总体影响

类型 1 块：2,352 个warp-phase中的 336 个具有控制分支 类型 2 块：392 个warp-phase中有 14 个具有控制分支 性能影响预计小于 12% (350/2,744 或 (336+14)/(2352+392))

额外的注释

加载 N 个tiles时控制分支的影响计算有些不同，留作练习 估计的性能影响取决于数据。

  • 对于较大的矩阵，影响将显着较小

一般来说，控制发散对大型输入数据集的边界条件检查的影响应该是 微不足道

  • 应该毫不犹豫地使用边界检查来确保完整的功能

内核中充满控制流结构的事实并不意味着会出现严重的控制发散

并行规约

规约和总结

处理大型输入数据集的常用策略

  • 数据集中的处理元素没有必需的顺序（关联和交换）

  • 将数据集分成更小的块

  • 让每个线程处理一个块

  • 使用归约树将每个块的结果汇总为最终答案

EG、谷歌和Hadoop 地图规约 框架采用这一战略 我们现在将专注于规约树步骤

什么是规约计算

总结 使用“归约操作”将一组输入值转换为一个值 最大，最小，和，乘积 常与用户定义的归约操作函数一起使用，只要操作是。。。

  • 是结合的和可交换的
  • 具有明确定义的标识值（例如，总和为 0）
  • 例如，用户可以为 3D 坐标数据集提供自定义的“max”函数，其中每个坐标数据元组的大小是距原点的距离。

一个有效的串行规约

将结果初始化为归约操作的标识值 最小值的用最大归约，最大值的用最小规约，0 用总和规约1 用积减少 遍历输入并执行结果值和当前输入值之间的归约操作

  • 每个 对 N 个输入值执行的归约操作
  • O(N) 算法
  • 这是一种计算效率高的算法。

并行归约树算法以 log(N) 步执行 N-1 次操作

快速分析

  1. 对于 N 个输入值，归约树执行 (1/2)N + (1/4)N + (1/8)N + … (1/N)N = (1- (1/N))N = N-1 个操作 在 Log (N) 步长中 – 1,000,000 个输入值需要 20 步 假设我们有足够的执行资源 平均并行度 (N-1)/Log(N)) 对于 N = 1,000,000，平均并行度为 50,000 但是，峰值资源需求为 500,000 这不是资源有效的

  2. 这是一个高效的并行算法 所做的工作量与高效的顺序算法相当 许多并行算法工作效率不高

  3. 我们称一个算法为高效算法(或者仅仅是高效算法)，如果它执行相同数量的工作，在一个常数因子内，作为已知的最快的顺序算法。

  4. Resource efficieny(Efficiency)=Sp/p=1,000,000/(20*500,000)=0.19(p=input/2)

如何用CUDA实现

并行求和规约

  • 并行实现
    • 每个线程添加 每个步骤中的两个值
    • 递归地将线程数减半
    • 对 n 个元素执行 log(n) 步，需要 n/2 个线程
  • 假设使用共享内存进行就地规约
    • 原始向量在设备全局内存中
    • 共享内存用于保存部分和向量
    • 最初，部分和向量只是原始向量
    • 每一步都使部分和向量更接近总和
    • 最终和将在部分和向量的元素 0 中
    • 由于读取和写入部分总和值，减少全局内存传输
    • 线程块大小限制 n 小于或等于 2,048

在计算机科学中，就地算法是一种不使用辅助数据结构转换输入的算法。

内存并行化

全局内存（DRAM）带宽

DRAM 核心阵列组织：

  • 每个 DRAM 核心阵列大约有 16M 位

4K*4K个存储位元组成。每个位元由一个晶体管和一个电容组成

Row 译码器；传感放大器；列锁存器；MUX 数据选择器

  • 每个位都存储在一个由一个晶体管制成的微型电容器中（1T1C）

一个非常小的（8x2 位）DRAM 核心阵列

DRAM 核心阵列很慢

从核心阵列中的单元读取是一个非常缓慢的过程 DDR：核心速度 = ½ 接口速度 DDR2/GDDR3：核心速度 = ¼ 接口速度 DDR3/GDDR4：核心速度=⅛接口速度 ……未来可能会更糟

DRAM 突发

对于时钟频率为接口速度 1/N 的 DDR{2,3} SDRAM 内核：

  • 从同一行一次加载（N × 接口宽度）DRAM 位到内部缓冲区，然后以接口速度分 N 步传输
  • DDR3/GDDR4：缓冲区宽度 = 8X 接口宽度

DRAM 突发时序示例

现代 DRAM 系统设计为始终以突发模式访问。 突发字节被传输到处理器，但当访问不是对连续位置时被丢弃。

DRAM突发-一个系统观点

每个地址空间被划分为突发部分

  • 每当访问一个位置时，同一部分中的所有其他位置也被传送到处理器

基本示例：16 字节地址空间，4 字节突发部分

  • 实际上，我们至少有 4GB 的地址空间，128 字节或更多的突发段大小

内存合并

当一个warp的所有线程都执行一条加载指令时，如果所有访问的位置都落入同一个突发部分，那么只会发出一个DRAM请求并且访问完全合并。

未合并的访问

当访问的位置跨越突发部分边界时：

  • 合并失败，发出多个 DRAM 请求，访问未完全合并。

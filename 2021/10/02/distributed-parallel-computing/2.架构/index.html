第二章：架构

冯·诺依曼模型的修改

冯·诺依曼模型：

  1. 冯诺依曼的核心是：存储程序，顺序执行。
  2. 计算机处理的数据和指令一律用二进制数表示；
  3. 指令和数据不加区别混合存储在同一个存储器中；
  4. 顺序执行程序的每一条指令；
  5. 计算机硬件由运算器、控制器、存储器、输入设备和输出设备五大部分组成。

冯·诺依曼瓶颈：内存和CPU的分离通常称为冯·诺伊曼（von Neumann）瓶颈。

冯·诺依曼模型的修改

  • 缓存
  • 虚拟内存
  • 低级并行

缓存基础

  • 缓存：可以比其他一些内存位置用更少的时间访问的内存位置的集合。

  • CPU高速缓存：CPU可以比其访问主内存更快地访问的内存位置的集合。CPU高速缓存可以与CPU位于同一芯片上，也可以位于与普通存储芯片相比访问速度更快的单独芯片上。

  • 关于缓存的想法

    • 使用更广泛的互连，并在单个内存访问中传输更多数据或更多指令。
    • 将数据块和指令块存储在实际上更靠近CPU寄存器的特殊内存中。

哪些数据和指令应该存储在缓存中？

程序倾向于使用以下数据和指令：物理上接近至最近使用的数据和指令。

1.地方性原则

  • 地点----访问一个位置之后访问附近的位置，访问一个内存位置后，程序将在里面不远的将来（时间局部性）访问一个附近地点（空间局部性）。
  • 为了利用局部性原理，系统使用了有效的更宽的互连以访问数据和指令。（Cache Block或者Cache Line）

最常见的形式，数组：

float z[1000]
    ....
sum =0.0
    for(i=0;i<1000;i++){
        sum+=z[i];
    }

2.缓存级别

目前大多数的计算机都有三级缓存。

L1的容量最小，但是最快。L2次之。L3的容量最大，但是最慢。

3.缓存命中与缺失

缓存命中：

此时需要使用x，在缓存中可以找到x。

缓存缺失：

此时需要使用x，在缓存中无法找到x，只好去内存中寻找。

4.缓存的写策略

当CPU将数据写入高速缓存时，高速缓存中的值可能与主存储器中的值不一致。

  • 写直达：在将数据写入缓存的同时，将数据写入主内存

  • 写回：当写发生时，新的数据只会被写入缓存块；只有当该数据替换时，才会写入下一级的缓存块

5.缓存的映射

  • 全相连映射：可以在缓存中的任何位置放置新行

  • 直接映射：每个高速缓存行在高速缓存中都有一个唯一的位置，它将被分配给该位置

  • n路组相连：每个高速缓存行可以放置在缓存中其中n路的不同位置。

一些问题：

1.当内存中的多行可以映射到缓存中的多个不同位置时，我们还需要能够决定哪一行应该被替代。

2.最常用的方案称为最近最少使用 。顾名思义，缓存中记录了使用块的相对顺序。

6.缓存和编程

了解空间和时间局部性的原理可以使我们对缓存进行一些间接控制。

double A[max][max],x[max].y[max]
    ...
/*Initialize A and x,assign y=0*/
    ...
/*first pair of loop*/
 for(i=0;i<max;i++){
     for(j=0;j<max;j++){
         y[i]+=A[i][j]*x[j];
     }
}
/*assign y=0*/
/*second pair of loop*/     
 for(j=0;j<max;j++){
     for(i=0;i<max;i++){
         y[i]+=A[i][j]*x[j];
     }
}

假设max=4。 假设A的元素都不在高速缓存中时 循环开始执行。 假设缓存行由A的四个元素组成，A [0] [0]是第一个缓存行的元素。 假设缓存为直接映射 而且只能存储A的八个元素或两条缓存行。

缓存加载时直接加载A数组的四个元素，访问A[0][0]时，缓存已经把紧随其后的 3 个元素也载入了。

第一次循环导致4次未命中，第二次循环导致16次未命中。

虚拟内存

  • 如果我们运行一个非常大的程序或访问一个非常大的数据集的程序，则所有指令和数据可能无法放入主存储器中。
  • 虚拟内存的开发使主存的功能成为二级存储的缓存。
  • 它利用了时空局部性的原理 ，仅将正在运行的程序的活动部分保留在主内存中

指令级并行

  • 指令级并行或ILP尝试通过以下方法来提高处理器性能： 多个处理器组件或者功能单元同时地执行指令。
    • 流水线----功能单元是分阶段安排的。
    • 多级发射问题----可以同时启动多个指令。

流水线

浮点数加法部件9.87×10^4 and 6.54×10^3

float x[1000],y[1000],z[1000]
    ..
for(i=0;i<1000;i++){
    z[i]=x[i]+y[i];
}

假设每个操作花费一纳秒（10^-9 秒），该for循环大约需要7000纳秒。

另一种方案：

  1. 将浮点加法器分为7个单独的硬件或功能单元。
  2. 第一个单元获取两个操作数，第二个单元比较指数，依此类推。
  3. 一个功能单元的输出输入到下一个功能单元。
  4. 可以在7种不同的添加方式中同时执行7个不同的阶段。
  • 一个浮点加法仍然需要7纳秒。
  • 但是现在1000个浮点数加法指令需要1006纳秒！
  • 通常，具有k个阶段的流水线不会获得k倍的性能提升。
    • 各个功能单元所需的时间不同。
    • 延迟可能导致管道停顿。

多级发射问题

多极处理器复制功能单元，并试图在一个时钟周期中发射多条指令

静态多级发射：功能单元是在编译时安排的

动态多级发射：功能单元在运行时进行调度。

为了利用多级发射，系统必须找到可以同时执行的指令。

在推测中，编译器或处理器对一条指令进行猜测，然后根据该猜测执行该指令。

硬件多线程

  • 并非总是有同时执行指令的好机会。

  • 线程级并行 尝试通过同时执行不同的线程来提供并行性。 它提供了一 比ILP 粗粒度 并行性。

  • 硬件多线程 提供了一种方法，使系统在当前正在执行的任务停止时继续进行有用的工作。

    例如，当前任务必须等待从内存加载数据。

细粒度：处理器在每条指令后在线程之间切换，跳过停滞的线程。 优点：避免因停顿而浪费机器时间的潜力。 缺点：准备好执行长指令序列的线程可能必须等待执行每条指令。

粗粒度 ：仅切换停滞的线程以等待耗时的操作完成。 优点：切换线程几乎不需要即时。 缺点：处理器可以在较短的停顿时间内空闲，并且线程切换也将导致延迟。

同时多线程 （SMT）- 细颗粒的多线程的变体 。  允许多个线程利用多个功能单元来利用超标量处理器。

并行硬件

Flynn分类法

  • SISD:单指令单数据流
  • SIMD:单指令多数据流
  • MISD:多指令单数据流
  • MIMD:多指令多数据流

1.SISD单指令单数据流

  • 一次执行一条指令，一次获取或存储一项数据
  • 冯·诺依曼系列机器的型号
  • 逻辑上，单控制处理器

2.MISD多指令单数据流

  • 多指令单数据
  • 该术语未使用（讨论Flynn分类法时除外）
  • 也许适用于流水线计算

3.SIMD单指令多数据流

  • 将同一条指令应用于多个数据项。
  • 单个控制单元和多个ALU
  • 多个处理器以锁步的方式执行相同的程序。

在“经典” SIMD系统中， 每个ALU 在继续之前必须等待下一条指令的广播。 所有ALU执行相同的指令或处于空闲状态。

  • 通过在处理器之间划分数据来实现并行性。每个处理器看到的数据可能会有所不同。

例1：

如果我们没有数据项那么多的ALU怎么办？

重复划分工作和流程，直至可以划分成功。

例2：m = 4个ALU 和 n = 15个数据项。

所有ALU执行相同的指令或处于空闲状态。它会降低SIMD系统的整体性能。

对于大型数据并行问题有效，但对其他类型的更复杂的并行问题则无效。 示例：矢量处理器

向量处理器

1.操作数组或数据向量，而传统CPU操作单个数据元素或标量。

2.最近的典型系统具有 具有以下特点：

1）向量寄存器。 能够存储操作数的向量并同时对其内容进行操作。

2）向量化和流水线功能单元。 将相同的操作应用于向量中的每个元素（或元素对）。

3）矢量指令。 根据向量而不是标量进行操作。

4）交错的内村

  • 多个 ”银行” 内存，可以或多或少地独立访问。
  • 在访问一个银行之后，在重新访问它之前会有一个延迟，但是访问另一个银行可以快得多。
  • 如果将一个向量的元素分布到多个行，可以减少或消除加载/存储连续元素的延迟。

5）交叉记忆访问和硬件分散/聚集

  • 该程序访问以固定间隔定位的向量的元素。
  • 分散/收集(在这种情况下)是在不规则的间隔中写入(分散)或读取(收集)向量的元素。

3.优点

  • 快速，易用。
  • 向量化编译器擅长识别可以向量化的代码。
  • 编译器还可以提供有关无法向量化的循环的信息并提供原因。帮助用户重新评估代码。
  • 高内存带宽。
  • 每个加载的数据项是 用过的。

4.缺点

  • 它们不处理不规则数据结构以及其他并行体系结构。
  • 他们处理更大问题的能力受到非常有限的限制。（可扩展性）

4.MIMD多数据多指令流

  • 支持多指令流同时执行在多数据流。
  • 通常由一组完全独立的处理单元或核组成，每个处理单元或核都有自己的控制单元和自己的ALU。
  • MIMD系统通常是异步的
  • 许多MIMD系统都有独立的时钟，示例：共享内存和分布式内存机器

MIMD物理组织

共享内存系统和分布式内存系统（MIMD）

关于细节方面，可以查看这篇博客：https://blog.csdn.net/ibless/article/details/80114009

1.共享内存系统（UMA也叫SMP和NUMA）

  • 自治处理器的集合通过互连网络连接到存储系统。
  • 处理器可以直接访问系统中的所有数据。
  • 处理器通常通过访问共享数据结构进行隐式通信。

多核处理器在一个芯片上有多个cpu或核心。

内核具有私有的一级缓存，而其他缓存可能在内核之间共享，也可能不共享。

在与多个多核处理器共享内存系统,直接互连可以连接所有处理器内存(UMA)或每个处理器可以直接连接到一块内存,处理器可以访问对方的主存块通过特殊硬件内置处理器(NUMA)。

2.统一内存访问（UMA）

所有内核访问所有内存位置的时间将相同。

3.非统一内存访问

我觉得博客中给的这个图片会更加直观：

在每个节点内部都有自己的内存，称之为本地内存。与本地内存相对的是远端内存，对于Node0来说，Node1、Node2、Node3中的内存都是远端内存。在节点中，CPU与内存之间通过片内总线进行连接。各个节点之间通过互联模块（Crossbar Switch）进行连接。需要注意的是，NUMA节点中CPU对节点内部内存与节点外部内存的访问是有差异的。依旧是一个CPU对本地内存和远端内存的访问时间是不同的，一般来说，CPU对本地内存的时间要比远端内存的访问速度快1.3--5倍。这也就是称为NUMA（非统一内存访问）的原因。NUMA既然属于共享内存架构，也就是每个CPU都可以访问系统中所有的内存，只不过访问这些内存的速度或方式是有差异的。一般来说，NUMA节点之间是相互连接的，这个有一个Node Distance的概念，也就是节点与节点之间的距离是不同的。

4.分布式内存系统

  • 每个处理器都有自己的专用内存

  • 处理器通过显式消息传递来共享数据或者特别的职能。

BUS interconnet（总线互联）

  • 一组并行通信线以及一些控制总线访问的硬件。
  • 通信线由与其连接的设备共享。
  • 低成本和灵活性
  • 随着连接到总线的设备数量的增加，使用总线的争用增加，性能降低。

Switched interconnect（交换互联）

  • 使用交换机控制连接设备之间的数据路由。

  • 交叉

    • 允许不同设备之间同时通信。
    • 比总线（BUS）快。
    • 但是交换机和链接的成本相对较高。

分布式内存互联网络

两组

  • 直接互连（静态网络） 每个交换机都直接连接到处理器-内存对，并且交换机彼此连接。 它由处理节点之间的点对点通信链接组成。
  • 间接互连 （动态网络） 交换机可能未直接连接到处理器。 它是使用交换机和通信链接构建的。

直接互联：

​ （a）环 （b）二维环面网格

如果有p个处理器，在环面网格链路的数目是3p，但在环中只是2p。

指标定义：

  • 等分宽度

    • 一种测量“同时通信的链路数目”或者“连接性”的方法

    • 并行系统分成两部分，每部分有一半的分水岭处理器或者节点，同时发生通信的次数

    • 等分宽度应该给出一个“最坏情况”的估计。

    • 计算平分宽度的另一种方法是去掉最少的链路数从而将节点分成两等分，删除的链接的数量是平分宽度。

如下图1，等分宽度为2，图2的等分宽度为4，但等分宽度应该为最坏的一种情况，所以是2而不是4。

如果有一个正方形的二维环面网格，有p=q^2（q为偶数）个节点，等分宽度为2√p。

  • 带宽

    • 链路可以传输数据的速率。

    • 通常以每秒兆位或兆字节为单位。

  • 等分带宽

    • 衡量网络的质量
    • 不是计算两个等分之间的链路数，而是计算链路的带宽，在环中，如果链路的带宽为10亿位每秒，那么环的等分带宽就是20亿位每秒。

  • 延迟和带宽

    • 消息传输时间 t=l+n/b

    • l：latency（延迟），从源开始发送数据到目标开始接收第一个字节之间经过的时间

    • n:消息的长度（bytes）

    • 带宽：目标开始接收第一个字节后接收数据的速率（每秒钟）

全相连网络：等分宽度为p^2/4，总共需要p^2/2+p/2条链路，切每个交换机都需要p条链路（不切实际的）

  • 每个交换机都直接与其他每个交换机相连。
  • 这是“理论上可能的最佳”互连， 但不切实际

线性网络：

  • 一维数组

    • 交换机排列成一维网格
    • 对应于二维网格的一行或一列

  • 环形

    • 允许在末端的交换机之间进行包络连接的变体。
    • 本质上支持双向管道

多维网格网络：

  • 网：

    • 将线性阵列推广到2D或3D

    • 仅允许相邻交换机之间的通信

  • 圆环面：

    • 包括网格边缘上的开关之间的环绕连接的变体（带有环绕的网格）

超立方体：

  • 超立方体是一个多维网格，每个维度上恰好有两个处理器。

  • 维度为d的超立方体具有p = 2^d 节点。

  • 递归的建立

    • 一维超立方体是由两个处理器的全互联系统
    • 二维超立方体是由两个一维超立方体组成，并通过“相应”的交换器互联
    • 三维超立方体是有两个二维超立方体组成，并通过“相应”的交换器互联
    • d维超立方体是有两个（d-1）维超立方体组成，并通过“相应”的交换器互联，每个处理器都与d个处理器进行互联。
    • 等分宽度为d/2，所以它比环或者环面网格连接线更高，但需要更大的交换器，每个交换器必须支持1+d=1+log2（p）连线

编号方案：

一个简单的编号方案可以从超立方体的构造推导出来。

  • 如果有p/2节点的两个子立方体的编号，通过在其中一个子立方体的标签前加上“0”，推导出p节点的立方体的编号方案;另一个子立方体的标签为“1”。

  • 这种编号方案有一个有用的特性，即两个节点之间的最小距离由两个标签中不同的位数给出。
    • 例如，标记为0110和0101的节点相隔两个链接，因为它们在两个位位置不同。
    • 这个特性对于推导超立方体体系结构的许多并行算法是有用的。

  • n=2^d的处理器节点，由n(logn+1）交换节点链接

Omega网络：

  • 交换器是一个2x2的交叉开关矩阵。

  • 注意与交叉开关不同的是，有一些通信无法进行，例如处理器0给处理器6发送一个消息，这是处理器1就不能同时给处理器7发送消息。

  • omega网络比交叉开关矩阵便宜，omgega网络使用了1/2plog2（p）个交换器，每个交换器是一个2x2的交叉开关矩阵交换器，所以总共使用了2plog2（p）个交换器。

  • 一个pxp大小的交叉开关矩阵的等分宽度是p，而一个omega网络的等分宽度是p/2。

Tree Network（树状网络）：

  • 树状网络是指任意一对节点之间只有一条路径的网络。

  • 静态树型网络在树的每个节点上都有一个处理元素。

  • 在一个 动态树网络，中间级别的节点是交换节点，叶节点是 处理元素。

    ​ （a）静态树网络 （b）动态树网络

  • 如何在树中路由？

    • 源节点向上发送消息，直到到达同时包含源节点和目标节点的最小子树的根节点。
    • 然后将消息沿着树路由到目标节点。

  • 树状网络在树的更高层次上遭受通信瓶颈。（对根节点的依赖性大：如果根节点发生故障，则整个网络都不能正常工作）

内存层次结构和缓存一致性

内存层次结构

  • 多级内存层次

  • 越靠近CPU的级别访问速度更大

  • 如果数据访问之间存在空间和时间局部性，则高速缓存存储器会很好地工作

  • 内存延迟

缓存一致性：

  • 共享内存的一个重要问题
  • 处理器可能会缓存相同的位置
  • 如果一个处理器写入该位置，所有其他处理器最终都必须看到该次写入

假设内存中有x变量的值1。 P1从内存加载x到自己的cache。 P2从内存加载x到自己的cache。 P1把自己的cache2的x修改为2，此时就产生了cache不一致的情况。 内存中的x就会被修改为2，P2的cache也被修改为2。来确保所有的处理器最终看到相同的x值

例子：

我们假设，y0这个变量被core 0私有。y1和z1被core1所私有。

x=2为共享变量。

接下来按照指定时序执行下面的语句:

time core 0 core 1
0 y0=x y1=3x
1 x=7 没有包含x的语句
2 没有包含x的语句 z1的4x

y0的最终值为2，y1的最终值为6。

但是z1的值是什么？

第一感觉可能是z=28，但是在时间0时，x=2已经在core 1的cache中了。除非出于某些特殊原因，x清除core 0的cache，然后重新加载到core 1的cache中；若没有上述情况，通常还是会使用原来的值为x=2，所以z=8。

这种不可预测的行为与写直达还是写回策略无关。如果采用写直达，内存中的x会因赋值而更新，但是这与core1的cache值无关。如果采用写回，那么更新z1的时候，x在core 0的cache中的新值对于core1还是不可用的。

程序员无法控制缓存以及何时更新缓存。

与共享数据有关的问题：

缓存一致性问题：

  • 跨多个缓存复制数据可减少处理器之间对共享数据值的争用。
  • 但是，我们如何确保不同的处理器对于相同的地址有相同的值呢?
  • 缓存一致性问题是当一个过时的值仍然存储在处理器的缓存中。
  • cache一致性保证的是在CPU访问到某个cache line时，其中的数据是和内存的对应位置，以及和其他CPU对应的cache line是一致的。

缓存一致性机制：

  • 更新vs无效
  • 基于监听的vs基于目录的（协议）

缓存一致性协议：

  • 更改变量的值时： 使所有副本无效还是更新呢？
  • 这两个协议都会产生伪共享（后面会有讲解）。

内存一致性（回忆一下）：

  • 写直达：当该行被写入缓存的同时，它被写入主存。
  • 数据不会立即写入。缓存中的更新数据被标记为旧值，当缓存线被内存中的新缓存线替换时，旧值被写入内存。

比较：

  • 写无效：当处理器写入本地缓存时，其他缓存的副本必须失效

    • 写直达：它还会更新内存中的副本
    • 写回：它使内存中的副本无效。

  • 写更新：当处理器写入本地缓存时，其他缓存的副本必须更新

    • 写直达：它还会更新内存中的副本
    • 写回：数据不会立即写入。缓存中的更新数据被标记为旧值，当缓存线被内存中的新缓存线替换时，旧值被写入内存。

  • 问题：成本效益权衡取决于流量模式

    • 什么时候invalidate更坏一点？

      一个生产者生产数据，多个消费者消费数据

    • 什么时候update更坏一点？

      1.在另一个CPU读取数据之前一个CPU进行多次写操作。

      2.缓存被不再读取的数据填满

使用写无效协议：

  • 现代计算机使用无效协议作为默认协议 （为什么？）

    具体实现细节可以查看这篇博客，https://zhuanlan.zhihu.com/p/95435168 ，原因部分放在下面：

    当某个CPU更改了自己的cache line中的共享变量的值时，其他也拥有这个共享变量的cache line拷贝的CPU会做什么？比如P2往cache line中的变量写入了13，那么P1会立刻更新自己cache line中该变量的值，将其也变成13么？

    假设过一会儿P2又将这个变量的值改为了18，照此逻辑，P1也应该立刻更新变量的值为18，但如果在此期间，P1根本就没有访问这个变量，那它的值是13还是18，根本就没有关系，更新岂不是做了无用功。

    在其他CPU对共享变量有写操作之后立刻更新的做法，被称为Write Update，这种方法逻辑简单，但效率不高，更好的做法是在其他CPU(P2)对cache line有写操作之后，(P1)仅仅是把自己的cache中对应的cache line标记为Invalid的状态，这种做法被称为Write Invalidate。

  • 数据项的每个副本都与一个状态相关联

  • 状态示例集:共享、无效或修改(MSI协议)

    • Shared：数据项的多个有效副本（写需要变为Modified，并且产生invalid）
    • Modified：仅存在一个副本（写不改变状态）
    • Invalid：数据副本无效的（读产生数据请求并且更新状态为shared）
    • 下图为其状态转换表（实线表示处理器动作，虚线表示一致性动作(总线监听动作)）

提示：这组状态和转换并不是唯一的。也可以定义其他状态和相关的转换。(如:MESI)，其中E为exclusive（独占的）

监听Cache一致性协议

  • 核心共享一个总线(不是必需的，但支持广播)。

  • 总线上传输的任何信号都可以被连接到总线上的所有核心“看到”

  • 当核心0更新存储在其缓存中的x副本时，它也会在总线上广播该信息。

  • 如果核心1正在“监听”总线，它将看到x已经被更新，它可以将x的副本标记为无效。

  • 如何将无效文件发送到正确的处理器?

    • 广播所有无效和读请求
    • 监听Cache监听并且在本地做出适当的一致性操作

  • 监听Cache的操作

    • 一旦数据被标记为dirty(modified)，所有后续操作都可以在本地缓存中执行，无需外部传输
    • 如果一个数据项被多个处理器读取，在所有缓存中转换到共享状态，所有随后的读操作都变成本地操作
    • 但是，如果多个处理器读取和更新相同的数据，就会出现一个基本的瓶颈问题，在总线上生成一致性请求，总线受带宽限制，每秒更新次数受到限制
    • 如果是写直达Cache，不需要额外的互联网络，因为每个核都能检测写；如果是写回Cache，需要额外的通信，因为对Cache的更新不会立即发送给内存。

  • 一致性的花费

    • 监听Cache：每个一致性的操作都要被发送给所有的处理器。
    • 为什么不将一致性请求只发送给那些需要被通知的处理器呢?（于是产生了基于目录的Cache一致性监听协议）
基于目录的Cache一致性协议

  • 基于目录：共享状态保存在目录中

  • 存在位：表示缓存块和缓存它们的处理器的位图。

例子：

x=1；

P0 P1
Time 0 load x load x
Time 1 write #3

1.当处理器P0和P1访问与变量x对应的块时，会发生什么？

当处理器 P0 和 P1 访问与变量 x 相对应的块时，块的状态更改为共享，并更新了存在位以指示处理器 P0 和 P1 共享块。

2.当P0写入变量时，会发生什么？

当 P0 在变量上执行存储时，目录中的状态更改为脏，P1 的存在位被重置。在处理器 P0 上执行的此变量的所有后续操作都可以在本地进行。

3.如果另一个处理器读取该值，将会发生什么？

如果其他处理器读取了该值，目录会注意到脏标签，并使用存在位将请求直接指向相应的处理器。处理器 P0 更新内存中的块，并将其发送到请求处理器。存在位将进行修改以反映此情况，状态更新为共享。

  • 基于目录方案的性能

    • 开销:状态更新的传播(通信开销);从目录生成状态信息(争用)
    • 如果一个并行程序需要大量的一致性操作(大量的读写共享数据块)，目录最终会限制它的并行性能
    • 存储目录的位可能会增加很大的开销-考虑扩展到许多处理器
    • 目录成为一个争论点--分布式目录方案是必要的

  • 分布式目录

    • 分布式目录方案的性能：比监听系统或集中式目录系统更具可扩展性
    • 允许O（p）处理器同时进行一致性运算
    • 底层网络必须承载所有一致性请求
    • 网络的延迟和带宽成为根本的性能瓶颈
伪共享

伪共享指的是不同处理器更新同一缓存线的不同部分的情况。 CPU缓存在缓存行而不是单个变量上运行。

具体细节可以参考这篇博客

串行版本：

int i,j,m,n;
double y[m];
/*Assign y=0*/
...
for(i=0;i<m;i++){
	for(j=0;j<n;j++){
        y[i]=f(i,j)
    }
}

并行版本：

/*private varibles*/
int i,j,iter_count;
/*shared varibles initialzed by one core*/
int m,n,core_count;
double y[m];
iter_count=m/core_count;
/*Core 0 does this*/
for(i=0;i<iter_count;i++){
    for(j=0;j<n;j++){
        y[i]=f(i,j);
    }
}
/*Core 1 does this*/
for(i=iter_count;i<2*iter_count;i++){
    for(j=0;j<n;j++){
        y[i]=f(i,j);
    }
}

假设共享内存系统有2个核，m = 8，y [0]存储在高速缓存行的开头，并且高速缓存行可以存储8个双精度数，而y占用一个完整的高速缓存行。

当核心0和核心1同时执行其代码时会发生什么？

因为y的所有值都存储在单个告诉缓冲行中，所有每次一个核执行语句y[i]=f(i,j),高速缓存行就会失效，当另一个核尝试执行该语句的时候由于自己的状态为invalid，就必须从内存中将更新过的高速缓存行取出。尽管核0和核1不会访问对方的元素，但大量的赋值操作会访问主存，这就是伪共享。

  • 伪的共享不会导致错误的结果。
  • 他能引起过多不必要的访存，降低程序的性能。

共享内存

  • 动态线程

    • 主线程等待工作，派生新线程，并在线程完成后终止。
    • 有效利用资源，但是创建和终止线程非常耗时。

  • 静态线程

    • 创建并分配工作的线程池，但是直到清理后才终止。
    • 更好的性能，但可能浪费系统资源。

  • 不确定性

    • 如果给定的输入可以导致不同的输出，则计算是不确定的。

情况1：

线程1> my_val = 19 线程0> my_val = 7

情况2：

线程0> my_val = 7 线程1> my_val = 19

my_val = Compute_val（my_rank）;

      shared int n = . . . ;
shared double x [ n ] , y [ n ] ;
private int i , my_first_element , my_last_element ;
my_first_element = . . . ;
my_last_element = . . . ;
/ * Initialize x and y  */
. . .
for ( i = my_first_element ; i <= my_last_element ; i++)
     x [ i ] += y [ i ] ;

      首先定义两个共享数组，x和y。如果分配的核同时拥有x和y的元素，那么执行速度是很快的，但是如果核分配的全为x或者y，那么程序的性能非常糟糕，因为每次都要访问自己远端的内存。

输入和输出

当我们的并行程序需要执行I / O时，请进行以下假设并遵循以下规则：

  • 在分布式内存程序中，只有进程0会访问stdin。在共享内存程序中，只有主线程或线程0将访问stdin。
  • 在分布式内存和共享内存程序中，所有进程/线程都可以访问stdout和stderr。
  • 但是，由于输出到stdout的顺序不确定，在大多数情况下，除了调试输出之外，所有输出到stdout的只有一个进程/线程。
  • 调试输出应该始终包括生成输出的进程/线程的级别或id。
  • 只有单个进程/线程将尝试访问除stdin、stdout或stderr之外的任何单个文件。例如，每个进程/线程可以打开自己的私有文件进行读写，但没有两个进程/线程会打开相同的文件。

CPU和GPU的设计非常不同

CPU：面向延迟的设计

GPU：面向吞吐量的设计

获胜的程序同时使用CPU和GPU

GPU用于吞吐量胜出的并行部分：对于并行代码，GPU 可比 CPU 快 10 倍以上CPU 用于延迟重要的串行部分：对于顺序代码，CPU 可比 GPU 快 10 倍以上

CUDA C vs. CUDA Libs vs.OpenACC

加速应用程序的3种方法

GPU加速库

图形处理器加速库为您的应用程序提供“插入式”加速，有各种各样的计算库可用。

  • 您可能熟悉 FFTW 快速傅立叶变换库、各种 BLAS 线性代数库、

  • Intel Math Kernel Library，或 Intel Performance Primitives 库，仅举几例。有兼容的 GPU 加速库可作为替代品对于我刚刚提到的所有库，以及更多。

  • cuBLAS 是 NVIDIA 的 GPU 加速 BLAS 线性代数库，其中包括非常快速的密集矩阵和向量计算。

  • cuFFT 是 NVIDIA 的 GPU 加速快速傅立叶变换库。

  • cuRAND 是 NVIDIA 的 GPU 加速随机数生成库，可用于蒙特卡罗算法和金融应用程序等。

  • cuSPARSE 是 NVIDIA 的稀疏矩阵库，其中包括稀疏矩阵向量乘法、快速稀疏线性方程求解器和三对角求解器。

  • NVIDIA Performance Primitives 包含数以千计的信号和图像处理功能。 Thrust 是一个开源的 C++ 模板库，包含排序、归约、搜索和集合操作等并行算法。还有更多来自第三方开发人员和开源项目的库。

GPU 加速库是将 GPU 加速添加到应用程序的最直接方式，在许多情况下，它们的性能是无与伦比的。

GPU编程语言

在本章中，将讨论各种可用的 GPU 编程语言和接口。这些语言包括 C、C++、C#、Fortran、Python 和数值分析包，例如 MATLAB、数学， 和 LabVIEW.

CUDA编程模型

1.编程模型 是抽象的 底层计算机系统 允许 算法 和 数据结构的表达.

2.语言 和API 提供这些抽象的实现，并允许将算法和数据结构付诸实践 —— 编程模型的存在与编程语言和支持 API 的选择无关。

编程模型弥合了底层硬件架构和应用程序可用的软件支持层之间的差距。 编程模型不同于编程语言和应用程序编程接口 (API)。

一些设计目标

  • 扩展到100个内核，1000个并行线程
  • 让程序员专注于并行算法 ，不是并行编程语言的机制。
  • 启用异构系统（即 CPU+GPU），CPU 和 GPU 是具有独立 DRAM 的独立设备

关键的并行抽象

  • 并发线程的层次结构
  • 轻量级的同步原语
  • 用于协作线程的共享内存模型

线程层次结构

thread（线程）：由 CUDA 运行时分发 （由threadIdx识别) warp（线程束）：线程的调度单元，一个warp包含32个并行thread，这些thread以不同数据资源执行相同的指令 block（块）：用户定义的 1 到 512 个线程组。 (由blockIdx) grid(网格)：一或多个块组成，网格为每个CUDA核函数所创立。

CUDA内存层次结构

CUDA 平台具有三种主要内存类型：

  • 寄存器：每个thread内存用于自动变量和寄存器溢出。
  • 共享内存：每个block具有低延迟内存以允许块内数据共享和同步。线程可以通过这个内存安全地共享数据，并且可以通过_ _syncthreads()函数执行障碍同步
  • 全局内存：device level可以在块或网格之间共享的内存

CUDA 设备内存管理 API 函数

cudaMalloc()

  • 在设备全局内存中分配一个对象
  • 两个参数：指向被分配对象的指针的地址；已分配对象的大小(以字节为单位)

cudaFree()

  • 从设备全局内存中释放对象
  • 一个参数：指向要释放对象的指针

HOST设备数据传输API函数

cudaMemcpy()

  • 内存数据传输
  • 需要四个参数：指向目的地的指针 ；指向源的指针；复制的字节数；转移类型/方向
  • 传输到设备是异步的

ISA级别的程序

1.程序是存储在内存中的一组指令，可由硬件读取、解释和执行。

  • CPU 和 GPU 都是基于（不同的）指令集设计的

2.程序指令对存储在存储器和/或寄存器中的数据进行操作。

作为冯·诺依曼处理器的线程

线程是“虚拟化的”或“抽象的” 冯-诺依曼处理器。

并行线程数组

CUDA 内核由线程组成的网格（数组）执行

  • 网格中的所有线程运行相同的内核代码 (SPMD)，单程序多数据)‏
  • 每个线程都有用于计算内存地址和做出控制决策的索引

线程块：可扩展的合作

将线程数组分成多个块：

  • 块内的线程通过 共享内存，原子操作 和 屏障同步写作
  • 不同块中的线程不交互

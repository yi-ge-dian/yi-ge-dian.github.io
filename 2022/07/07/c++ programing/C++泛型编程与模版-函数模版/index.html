C++泛型编程与模版-函数模版

相关概念

模版

模版是将一个事物的结构规律予以固定化、标准化的成果，它体现的是结果形式的标准化。

函数模版

函数模版不是一个实在的函数，编译器不能为其生成可执行代码。定义函数模版后只是对一个函数功能框架对描述，当它具体执行时，将根据传递的实际参数决定其功能 。

类模版

类模版是一个类家族的抽象，它只是对类的描述，编译程序不为类模版（包括成员函数定义）创建程序代码，但是通过类模版的实例化可以生成一个具体的类以及该具体类的对象。

泛型编程

具体在多种数据类型皆可操作的含义，与模版有些类似。泛型编程是一种思想，模版是一个方法。

标准库

C++中标准程序库是类库和函数的集合。不同的编译器或不同的操作系统，实现方式可能会有一些不一样。

STL

标准模版程序库是C++标准程序库的子集，包含容器、算法、迭代器、函数对象等等。

函数模版定义

格式一般为：template<class T>

  1. 函数模板实际就是写一个通用函数，返回值和参数的类型都可变的，用一个特定格式的变量来指定。
  2. 然后调用此函数的时候，编译器会根据参数的数据类型，自动推导类型，从而达到函数在不同的类型参数下可复用的目的。
  3. 这个通用函数，就是函数模板。
  4. T是一个变量名，不是固定非要用T，写T是一个编程习惯，除了用class关健字之外，还可以用 typename，参数也可以有多个如 T1,T2。

函数模版的调用和参数推导

funcT1(a,b) ：这种是编译器根据参数推导类型

funcT1<>(a,b)：编译器根据参数推导类型，用来指明传递类型。

可以看到，编译器在执行的时候只是把对应的函数替换了，过程称为实例化。

如果函数定义完了，但是没有调用，编译器会优化掉，同理，如果一个参数没有被使用，也会被编译器优化。

函数模版的重载

函数模板与普通函数的重载：调用规则同时满足普通函数与函数模板时优先调用普通函数。

函数模板与函数模板的重载：智能判断适合自己的函数。比如传地址，就会走参数为指针那个函数。

当调用函数模板时，如果函数体内不支持类型的运算，会报错。

函数模版的泛化与特化
